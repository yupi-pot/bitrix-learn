# tools/

CLI-скрипты для работы с Битриксом без админки.

## Как запускать

Из корня репо:

```bash
./tools.sh <имя-скрипта>       # без .php
./tools.sh                     # список доступных
```

Под капотом — `docker exec bitrix_web php /var/www/html/tools/<script>.php`.

## Скрипты

| Скрипт | Что делает |
|---|---|
| `db-info` | Печатает все инфоблоки, свойства, HL-блоки, группы пользователей и значения констант из `init.php`. Первая команда когда надо разобраться "а что у нас в базе". |
| `clear-cache` | Чистит файловый + managed + tagged кеш. Запускать после любых правок кеширующихся компонентов, меню, прав. |

## Добавить новый скрипт

1. Создать `www/tools/новое-имя.php`
2. Первой строкой `require __DIR__ . '/_bootstrap.php';` — это подгрузит ядро Битрикса
3. Запускать через `./tools.sh новое-имя`

Файлы, начинающиеся с `_`, в списке не отображаются (служебные).

## Почему не Битриксовая `bitrix.cli`

Штатной CLI у Битрикса нет, но ядро прекрасно грузится через `prolog_before.php`
из любого PHP-файла. `_bootstrap.php` делает ровно это — подменяет серверные
переменные и тянет пролог. Дальше можно использовать любой API: `CIBlock::Add`,
`HighloadBlockTable::add`, `CUser::Add` — всё как на обычной странице.
